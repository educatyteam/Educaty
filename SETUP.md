# Educaty — настройка передачи лидов

Форма мастер-класса отправляет данные одним JSON-запросом на ваш **lead-forwarder** (серверная "прослойка"). Прослойка раскладывает заявку в Telegram-чат и в amoCRM. Это нужно, потому что:

- из браузера нельзя безопасно ходить в Telegram Bot API: токен бота светится в исходниках сайта;
- amoCRM webhook принимает POST, но CORS у него обычно прибит, поэтому ходить из браузера тоже неудобно;
- одна прослойка = один URL в коде сайта, остальное настраивается без правки фронтенда.

Готовый код прослойки лежит в файле **`lead-forwarder.js`** (Cloudflare Worker, бесплатный тариф).

---

## 1. Telegram-бот для уведомлений

1. В Telegram найдите **@BotFather** → команда `/newbot` → задайте имя.
2. Получите токен вида `7123456789:AAEhBOweik6ad9r_QXrGcDvFOlVoMQabcdef`. Сохраните.
3. Добавьте бота в нужный чат / канал и **выдайте ему права писать**.
4. Узнайте `chat_id` чата:
   - Самый простой способ — добавить в чат `@RawDataBot` или `@getidsbot`, написать любое сообщение, бот пришлёт ID. У супергрупп ID обычно начинается на `-100…`.
   - Альтернативно: открыть `https://api.telegram.org/bot<TOKEN>/getUpdates` после того, как кто-то написал в чат.

> Примечание: те ID, которые видны в админке Tilda («Telegram: Education» → ID и Secret) — внутренние для Tilda и здесь не подходят. Тут нужен **бот, созданный вами**.

---

## 2. amoCRM — входящий вебхук

1. Зайдите в amoCRM → **Настройки → Интеграции → API → Входящие вебхуки** (или: Воронка → ⋯ → Robotic actions → Webhook на этап).
2. Создайте webhook **«Создавать сделки/контакты из формы»**.
   - Источник: «Создание контактов и сделок».
   - URL копировать ваш не нужно — наоборот, amoCRM даст вам **свой URL** вида `https://your.amocrm.ru/api/v4/leads/...` или `https://your.amocrm.ru/incoming/leads/form/<hash>`.
3. Этот URL сохраните — это и есть `AMOCRM_WEBHOOK_URL` для прослойки.
4. В правилах сопоставления полей привяжите поля формы:
   - `contact[name]` → Имя контакта;
   - `contact[phone]` → Телефон контакта;
   - `lead[name]` → Название сделки;
   - `lead[tags]` → Тег.

> Если в вашем тарифе нет «входящих вебхуков», можно использовать **`Salesbot`** или ставить **`API ключ + интеграция`** — но тогда придётся OAuth. В 95% кейсов «Входящий вебхук формы» закрывает задачу.

---

## 3. Разворачиваем прослойку (Cloudflare Worker, ~3 минуты)

1. Откройте https://dash.cloudflare.com → **Workers & Pages** → **Create application** → **Create Worker**.
2. Назовите, например, `educaty-leads`. Нажмите **Deploy** (создастся пустой воркер).
3. Откройте созданный воркер → **Edit code**.
4. Удалите дефолтный код. Скопируйте всё содержимое **`lead-forwarder.js`** из этой папки → вставьте в редактор.
5. **Save and Deploy**.
6. Перейдите **Settings → Variables → Add variable** и добавьте:
   - `TELEGRAM_BOT_TOKEN` — токен из BotFather.
   - `TELEGRAM_CHAT_ID` — id чата (например `-1001942201905`).
   - `AMOCRM_WEBHOOK_URL` — URL из amoCRM (можно пропустить, если пока не нужно).
   - `ALLOWED_ORIGIN` — `https://educaty.ru,https://www.educaty.ru` (или `*` на первое время).
7. Скопируйте URL воркера сверху страницы (вида `https://educaty-leads.<user>.workers.dev`).

---

## 4. Связываем фронт с прослойкой

В `index.html` найдите строку:

```js
const LEAD_ENDPOINT = "https://YOUR-LEAD-FORWARDER-URL.workers.dev";
```

и впишите туда URL вашего воркера. Сохраните, опубликуйте. Готово.

---

## 5. Размещение в Tilda

Поскольку Tilda-интеграцию не делаем, есть три варианта:

**Вариант A — целиком как одностраничник на Tilda «Code injection»:**

1. В Tilda создайте Zero Block или подключите блок T123 (HTML).
2. Скопируйте `<style>…</style>` из `<head>` лендинга в **Настройки сайта → Дополнительно → HTML-код в HEAD страницы**.
3. Скопируйте всё содержимое `<body>…</body>` (без самого `<body>`) в блок T123.
4. Шрифты Google Fonts добавьте в HEAD-секцию (`<link>`-теги из лендинга).
5. Поднимите файлы `assets/` на CDN (любой — Tilda, Cloudflare R2, ваш сервер) и поправьте пути.

**Вариант B — на отдельном домене (рекомендую):**

1. Захостите файлы как есть на Cloudflare Pages / Vercel / Netlify (просто `git push`).
2. Прицепите домен.
3. Форма уже знает, куда отправлять (LEAD_ENDPOINT).

**Вариант C — как iframe внутри Tilda:**

1. Захостите лендинг на отдельном домене (Cloudflare Pages, бесплатно).
2. В Tilda вставьте блок T123 с `<iframe src="https://landing.educaty.ru" width="100%" height="100%"></iframe>`.
3. Это самый ленивый путь, но Tilda-навигация перестанет работать поверх лендинга — годится только если лендинг сам по себе и сидит на отдельной странице.

---

## 6. Проверить, что всё работает

1. Откройте лендинг, заполните форму, нажмите «Подать заявку».
2. В чат должна прийти заявка в течение 2 секунд.
3. В amoCRM в воронке появится новая сделка с тегом `ai-creator,landing`.
4. Если что-то не приходит:
   - Cloudflare Workers → ваш воркер → **Logs (Live)** — увидите трейс запроса и ошибки.
   - Проверьте, что переменные окружения проставлены.
   - Проверьте, что в `ALLOWED_ORIGIN` есть домен, с которого открывается лендинг.
